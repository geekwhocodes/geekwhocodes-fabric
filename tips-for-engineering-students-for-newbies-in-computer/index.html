    Tips for Engineering Students | For Newbies in Computer & IT

    Posted on 07 Jul 2016

    Geekwhocodes
    tweets as @geekwhocodes

    I've noticed that most of the engineering students have problem "How or Where to start with Programming?". So in this post you'll find some easy tips to overcome those silly excuses.

    One thing to remember :

    Any fool can write code that computer can understand. Good programmers write code that humans can understand. ~Martin Fowler

     

    1. Use Search Engines very aggressively1 -

         Learn how to ask your questions in right way. there is an awesome book out there "Google For Dummies by Brad Hill", in that book he explained everything you need to know about finding the stuff you want without wasting too much time. you can find most of your answer's on stackoverflow.com.

    2. Don't count number of hours -

         I found that many students does count how many hours they were coding but you need to spend your time doing “deep work” (as opposed to “shallow work” I read this article it's great), and understand that the number of hours spent working/coding don’t actually matter. Note : There is a massive disconnect between what is taught in universities and what is done on a daily basis at startups(I'm experiencing this everyday here at RapidCircle).

    3. Spend more time on analysing the problem -

         This is the big mistake newbie does every time(I used to do this but thanks to Omkar, he helped me there.), traditional education has taught us that failure is bad(some of us thinks that teacher's code is the only way to solve this/particular problem and students copy each and every line as it is without any knowledge - don't do it just for the sake of completion, ask questions if you don't understand it.). if get an error messages, don't freak out these messages are actually the clues that lead you down the path to the right solution.

    4. Check your code twice before giving up or start blaming others -

         If your code is not working as you expect, some of us(count me in) blame others(like who developed this language and why etc. :P) or just gave up(I can't do this), however check your code twice.

    5. Learn by Emulation -

         Emulate the best, and this is possible by reading books written by the best and/or going through code from some of the best free software projects. I'll recommend "How to Solve it by Computer" book(only for newbies, it explained very basic concepts in easy approach). if you are not fan of books then make Github your best friend.

    6. Don’t dismay yourself by looking at changing technology world -

         Some of us get disappointing/discouraged, when they found that technology is moving very fast and I can't even run "Hello World" program. remember it's never too late start so you just need to "Start Somewhere".

    7. You'll learn by helping others -

         Personally I experienced this(I taught some of my friends and every time I learnt something new). discuss programming concept in group, do pair programming. I'll be happy to help you (just drop me an email here).

    8. Acknowledge that micro-decisions matter -

         example – naming variables, calling functions, naming CSS properties, and other seemingly small things can have a big impact.

    9. Don't compare yourself with other's -

         Your comparison of yourself with others will only result in evolution of negative feelings and un-healthy competition(happened with me even I always wanted to help everyone).Everyone has got his or her strengths and weaknesses. It is more important that we understand ours and work on it. I have seen many times that so called ‘fundoo-programmers’ (fundamentally strong programmer) also make silly mistakes. So, analyse yourself, list down your areas of improvement and work on it. "Programming is a real fun, enjoy it".



    1. if you don't know what is search engines, please change your branch as soon as possible. ↩
